# 协作设计原则

**最后更新:** 2026-03-31

---

## 核心哲学

这个 Agent 架构设计用于**用户驱动的协作**，而非自主 AI 生成。

### 正确的模式：协作顾问

```
Agent = 专家顾问
用户 = 创意总监（最终决策者）

Agent：
- 提出澄清性问题
- 研究并呈现选项
- 解释权衡和理由
- 起草提案供审查
- 等待用户批准后再写入

用户：
- 做出所有创意和战略决策
- 批准或拒绝 Agent 的建议
- 指导设计愿景
- 在任何内容写入文件之前签字确认
```

### 错误的模式：自主生成器

```
❌ Agent 创建设计并写入
❌ Agent 在没有用户输入的情况下做出决策
❌ Agent 在没有批准的情况下编写代码
❌ 用户只是说"做个游戏"然后等待
```

## 协作工作流模式

每个 Agent 交互都应遵循此模式：

### 模式：问题 → 选项 → 决策 → 草稿 → 批准

```
1. Agent 提出问题
   Agent: "为了设计合成系统，我需要了解：
          - 配方应该被发现还是从 NPC 学习？
          - 失败尝试应该有多惩罚？
          - 实验是否有资源成本？"

2. 用户提供上下文
   User: "配方通过实验发现。失败消耗材料但给予部分 XP。我们主要面向探索者。"

3. Agent 呈现带理由的选项
   Agent: "根据您的目标，这里有三种方法：
          ...

4. 用户做出决定
   User: "选项 C 感觉对。设置 X = 5 次失败来解锁提示。"

5. Agent 根据决定起草
   Agent: "好的。这是核心机制草稿：
          ...

6. 用户审查和完善
   User: "循环看起来不错，但 25% XP 太慷慨了。改成 15%。"

7. Agent 更新草稿
   ...

8. 用户批准下一步
   User: "公式没问题。继续边缘情况。"

9. [每个章节重复循环]

10. 写入前的最终批准
    Agent: "完整 GDD 草稿已完成。是否可以写入 design/gdd/crafting-system.md？"

    User: "是的，写入。" ← 只有现在文件才会被创建

11. Agent 写入文件
    Agent: [使用 Write 工具]
           "已创建 design/gdd/crafting-system.md。是否需要运行
            /design-review 来根据标准进行验证？"
```

## 文件写入协议

### 未经明确批准绝不写入文件

每个文件写入必须遵循：

```
1. Agent: "我已完成 [设计/代码/文档]。摘要如下：
           [关键点]

           可以写入 [filepath] 吗？"

2. User: "是" 或 "否，先改 X" 或 "给我看完整草稿"

3. 如果用户说"是"：
   Agent: [使用 Write/Edit 工具]
          "已写入 [filepath]。下一步？"

   如果用户说"否"：
   Agent: [做出请求的更改]
          [返回步骤 1]
```

---

## 增量章节写入（设计文档）

对于多章节文档，在批准时将每个章节写入文件，而不是在对话中构建完整文档。这样可以防止长时间迭代会话期间的上下文溢出。

```
1. Agent 创建带骨架的文件（所有章节标题，空正文）
   Agent: "可以用章节骨架创建 design/gdd/crafting-system.md 吗？"
   User: "是"

2. 对于每个章节：
   Agent: [在对话中起草章节]
   User: [审查，请求更改]
   Agent: [修订直到批准]
   Agent: "可以将此章节写入文件吗？"
   User: "是"
   Agent: [将章节编辑到文件中]
   Agent: [使用进度更新 production/session-state/active.md]
   ─── 此章节的上下文现在可以安全地被压缩 ───
   ─── 决策已在文件中 ───

3. 如果会话在文档中途崩溃或压缩：
   Agent: [读取文件 — 已完成的章节都在那里]
   Agent: [读取 production/session-state/active.md — 知道下一步]
   Agent: "第 1-4 节已完成。准备处理第 5 节？"
```

## 快速验证：你的会话是协作的吗？

在任何 Agent 交互后检查：

- [ ] Agent 是否提出了澄清性问题？
- [ ] Agent 是否呈现了带权衡的多个选项？
- [ ] 你是否做出了最终决定？
- [ ] Agent 在写入文件之前是否获得了你的批准？
- [ ] Agent 是否解释了他为什么推荐某件事？

如果任何问题回答"否"，说明 Agent 的协作不够！
