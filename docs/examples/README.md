# 协作会话示例

此目录包含真实的端到端会话记录，展示游戏工作室 Agent 架构在实践中如何工作。每个示例演示了**协作工作流**，其中 Agent 提出问题、呈现选项并等待用户批准，而不是自主生成内容。

---

## 可用示例

### [会话：设计合成系统](session-design-crafting-system.md)
**类型：** 设计
**Agent：** game-designer
**时长：** 约 45 分钟（12 轮）
**复杂度：** 中等

**场景：**
独立开发者需要设计一个服务于支柱 2（"通过实验的涌现发现"）的合成系统。Agent 通过问答引导他们，呈现 3 个带游戏理论分析的设计选项，纳入用户修改，并在每个步骤迭代起草 GDD。

**关键协作时刻：**
- Agent 提前提出 5 个澄清问题
- 呈现 3 个不同的选项，带优缺点 + MDA 对齐
- 用户修改推荐选项，Agent 立即纳入
- 主动标记边缘情况
- 每个 GDD 部分在进入下一部分前显示供批准
- 创建文件前明确 "我可以写入 [文件] 吗？"

### [会话：实现战斗伤害计算](session-implement-combat-damage.md)
**类型：** 实现
**Agent：** gameplay-programmer
**时长：** 约 30 分钟（10 轮）
**复杂度：** 低-中等

**场景：**
用户有完整的设计文档，想要实现伤害计算。Agent 阅读规格，识别 7 个歧义/差距，提出澄清问题，为批准提出架构，在规则执行下实现，并主动编写测试。

**关键协作时刻：**
- Agent 首先阅读设计文档，识别 7 个规格歧义
- 在实现之前提出带代码示例的架构提案
- 用户请求类型安全，Agent 完善并重新提案
- 规则捕获问题（硬编码值），Agent 透明修复
- 遵循验证驱动开发主动编写测试
- Agent 提供下一步选项而不是假设

### [会话：范围危机 - 战略决策](session-scope-crisis-decision.md)
**类型：** 战略决策
**Agent：** creative-director
**时长：** 约 25 分钟（8 轮）
**复杂度：** 高

**场景：**
独立开发者面临危机：Alpha 里程碑还有 2 周，合成系统需要 3 周，投资者演示是成败关键。创意总监收集上下文，构建决策框架，呈现 3 个带诚实权衡分析的战略选项，做出推荐但推迟给用户，然后用 ADR 和演示脚本记录决策。

**关键协作时刻：**
- Agent 在提出解决方案前阅读上下文文档
- 提出 5 个问题以了解决策约束
- 正确构建决策（什么是风险，评估标准）
- 呈现 3 个带风险分析和历史先例的选项
- 做出强烈推荐但明确："这是你的决定"
- 记录决策 + 提供支持用户的演示脚本

## 这些示例演示什么

所有示例都遵循**协作工作流模式：**

```
问题 → 选项 → 决策 → 草稿 → 批准
```

### 协作行为示例：

1. **Agent 在假设之前提问**
   - 设计 Agent 询问目标、约束、参考
   - 实现 Agent 澄清规格歧义
   - 领导 Agent 在推荐前收集完整上下文

2. **Agent 呈现选项，不是命令**
   - 2-4 个带优缺点的选项
   - 基于理论、先例、项目支柱的推理
   - 做出推荐，但由用户决定

3. **Agent 在定稿前显示工作**
   - 逐节显示设计草稿
   - 在实现前显示架构提案
   - 在决策前呈现战略分析

4. **Agent 在写入文件前获得批准**
   - 在使用 Write/Edit 工具前明确 "我可以写入 [文件] 吗？"
   - 多文件更改首先列出所有受影响文件
   - 用户说 "是" 然后才创建任何文件

5. **Agent 迭代反馈**
   - 用户修改立即纳入
   - 当用户更改推荐时没有防御性
   - 当用户改进 Agent 建议时表示赞赏

## 如何使用这些示例

### 对于新用户：
在第一次会话之前阅读这些示例。它们展示了 Agent 工作的真实期望：
- Agent 是顾问，不是自主执行者
- 你做出所有创意/战略决策
- Agent 提供专家指导和选项

### 对于理解特定工作流：
- **设计系统？** → 阅读 session-design-crafting-system.md
- **实现代码？** → 阅读 session-implement-combat-damage.md
- **做出战略决策？** → 阅读 session-scope-crisis-decision.md
